# Инструкции по разработке проекта (Landing Page с "вкусным" UI)

## 1. Стек и Архитектура
- **Фреймворк:** Next.js 14+ (App Router), TypeScript, Tailwind CSS.
- **Компоненты:** Только функциональные компоненты React с полной типизацией пропсов (`interface Props`).
- **Локализация:** Весь интерфейс, системные сообщения, ошибки валидации и подсказки должны быть СТРОГО на русском языке.

## 2. Качество UI/UX и Стилизация
- **Адаптивность (Mobile-First):** Любой элемент, сетка (Grid) или интерактивный блок должны идеально перестраиваться на мобильных разрешениях (до `md:`) без горизонтального скролла.
- **Премиальный дизайн:** Использовать современные паттерны (эффект размытия стеклянных поверхностей `backdrop-blur`, мягкие тени `shadow-xl`, плавные переходы `transition-all duration-300`).
- **Шрифты и Текст:** Адаптивные размеры текста (например, `text-2xl md:text-5xl`). Использовать семантические теги `<h1>`, `<h2>`, `<p>` правильно.

## 3. Анимации (Framer Motion)
- **Плавность:** Все анимации должны быть мягкими, без резких рывков. Рекомендуется использовать `type: "spring", stiffness: 100, damping: 15`.
- **Контроль DOM:** При анимации исчезновения элементов (например, переключение шагов квиза) обязательно использовать `<AnimatePresence mode="wait">`, чтобы разметка не прыгала.
- **Оптимизация:** Не перегружать анимациями мобильные устройства. Использовать `whileHover={{ scale: 1.02 }}` вместо агрессивных трансформаций.

## 4. Калькулятор и Квиз (Логика состояния)
- **Форматирование чисел:** Любые денежные значения должны форматироваться через функцию разделения тысяч (например, `1 250 000 ₽`).
- **Валидация форм:** Все интерактивные формы (в модалке калькулятора и финале квиза) должны иметь обязательную валидацию (маска телефона, проверка формата email, блокировка кнопки отправки при невалидных данных).
- **Чистота событий:** Предотвращать всплытие событий (`e.stopPropagation()`) в модальных окнах и оверлеях, чтобы избежать багов с закрытием/открытием интерфейса.

## 5. Проверка качества
- Перед каждым отчетом запускать локальную сборку `npm run build`. 
- Не оставлять "заглушек", неиспользуемых импортов (особенно иконок из lucide-react) и консоль-логов.
